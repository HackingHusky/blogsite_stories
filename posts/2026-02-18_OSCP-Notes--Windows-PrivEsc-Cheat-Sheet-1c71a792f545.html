OSCP Notes, Windows PrivEsc Cheat Sheet

So Windows PrivEsc can be really tricky. It’s largely the same as Linux, but you have to know what to look for. This excites me because you can do both in AD and Windows, and they go hand in hand. Especially with the AD chain you are being tested on. For these notes, I want to focus on initial access. AD will be its own section. If you get stuck during AD with a compromised machine, try these methods as well.

So the workflow of Windows PrivEsc:

                          +--------------------------------+
                          |     Windows Privilege Levels    |
                          |   (User → Admin → SYSTEM)       |
                          +--------------------------------+
                                          |
                                          v
        +----------------------------------------------------------------+
        |                  Common Security Exposure Areas                 |
        +----------------------------------------------------------------+
        |                          |                     |                |
        v                          v                     v                v
+----------------+      +-------------------+   +----------------+   +----------------+
| System         |      | Permission / ACL  |   | Credential     |   | Software /    |
| Configuration  |      | Misconfigurations |   | Weaknesses     |   | Patch Issues  |
+----------------+      +-------------------+   +----------------+   +----------------+
        |                          |                     |                |
        v                          v                     v                v
+----------------+      +-------------------+   +----------------+   +----------------+
| Services &     |      | Weak Folder/File  |   | Mismanaged     |   | Vulnerable     |
| Registry Setup |      | Permissions       |   | Tokens/Secrets  |   | Apps/Drivers   |
+----------------+      +-------------------+   +----------------+   +----------------+

It's the same approach as Linux: manual enumeration first, then WinPEAS. Winpeas is great. However, it will load you with all the info you can receive. You will likely need to bring your tools, such as PowerUp and AccessChk. Now you can run Mimikatz, getspn, or secretdumps; however, I am saving that for the AD cheatsheet section as I build these out. Both AD and Windows PrivEsc can be used for lateral movement. The goal here is that when you run into a Windows box, you have a checklist and commands on that box.

☐ Look for anything non‑default or unusual
☐ Take notes on every potential misconfiguration even if unsure

Honestly, there is much more to PrivEsc for Windows. If you stick to the same concept as Linux PrivEsc, the process is similar; you can also leverage AD attacks. RDP and WINRM are still great options, since you can get in, test it with accounts you intercept, and you're most likely to be in. Remember, as well, with the OSCP, it's most likely to be something goofy. So take your time.
