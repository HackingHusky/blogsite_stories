So an upload attack can be used. The password that was mentioned was 'admin admin', and I was able to access the service. The webpage didn't open for me until 5–10 minutes later. That's even adding the IP address and the hostname to the hosts file. After the website was loaded, I was able to access a reverse shell and retrieve the Ivan payload.

Obtained the reverse shell and stabilized it. Oddly, the first flag was in the www directory:

So, through privilege escalation, I manually found out the Python version and noticed that Python was using version 2, which ultimately allowed me to gain root access:

This is a fun, quick box from admin/admin, upload, to root. Sometimes Python does it for you—still a good one to try, and it gets the brain flowing. Migration: Don’t use default passwords and restrict www-data users' access.

For the study, uploading attacks are straightforward to execute; now you may have to manipulate the .htaccess file or create a malicious .pdf/Word document. However, once you have a user or upload to the web server, you can easily obtain a shell. That's one of my favorite attacks. Regardless of whether you Pentest Monkey or Ivan’s script, it's pretty straightforward.
