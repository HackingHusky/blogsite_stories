# Exploit Title: Remote Mouse GUI 3.008 - Local Privilege Escalation
# Exploit Author: Salman Asad (@deathflash1411) a.k.a LeoBreaker
# Date: 17.06.2021
# Version: Remote Mouse 3.008
# Tested on: Windows 10 Pro Version 21H1
# Reference: https://deathflash1411.github.io/blog/cve-2021-35448
# CVE: CVE-2021-35448

Steps to reproduce:

1. Open Remote Mouse from the system tray
2. Go to "Settings"
3. Click "Change..." in "Image Transfer Folder" section
4. "Save As" prompt will appear
5. Enter "C:\Windows\System32\cmd.exe" in the address bar
6. A new command prompt is spawned with Administrator privileges

According to the steps:

Now what you went to do is press enter, you don’t need to save it. But I saved it, restarted the service, and redid the attack mentioned above from the exploit database.

And the last flag has been captured. I definitely felt a few rabbit holes for a second until I got on with RDP, then checked who had access to that, and then ran the local exploit for privilege escalation. I had my total scropio moment from The Simpsons:

The software that ran the exploit - why didn't I think of that?!

Remediation for this attack: Update the software as soon as possible and filter ports TCP and UDP 1978—this is a simple fix.

This was a fun box; it's likely the same software exploit that allows for admin access. And that's one tricky part of the OSCP, or any Penetration Testing exam. You could easily fall down the wrong path and waste time running scripts like LinPeas or WinPeas trying to find that exploit. Even manual exploits have their problems, too. But take that moment; once you're in, trust your gut and ask, 'Does the same exploit have a local privilege escalation?' It could. It does come with muscle memory; the more you practice, the more beneficial it is to start from the boot to the root as well.
