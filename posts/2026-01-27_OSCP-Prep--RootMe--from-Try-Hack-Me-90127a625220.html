I like to know what results from whatweb too to check for different attacks. Whatweb is a Ruby-based web reconnaissance tool that identifies the technologies powering websites, including CMS platforms (WordPress, Joomla), blogging platforms, analytics tools, JavaScript libraries, and web servers. It's already built into Kali Linux. I sometimes use that to get a command-line view of the Technology, then I use Wappalyzer. Both are excellent tools to discover what the website is doing.

Now that I have an idea of the server, I usually run this gobuster command or use dirsearch:

gobuster:
gobuster dir -u http://$target -w /usr/share/wordlist/dirb/common.txt -t 40 -o results.txt

dirsearch:
dirsearch -u http://$target

How to install dirsearch:
sudo apt install dirsearch -y

I prefer dirsearch for the OSCP because it uses a default word list; however, you can add different word lists to your search, but that is my go-to for the OSCP.

Now I wanted to take this a step further and create a real-world scenario for this lab and write a mock report:

https://github.com/HackingHusky/CTF-Writeups/blob/main/RootMe%20Pentesting%20Report.pdf

It's about 14 pages long, but here is what to keep in mind for reports: the ability to translate technical speak into business-impact issues, along with their resolutions.

In the evidence field, you can be descriptive about technical know-how since the next tester or SOC analyst will look at the how, but you want to know your audience/stakeholders when they read this. This is also the role of a Pentester: not only to look for security holes, but also to translate them to stakeholders. Now, if this is going to the security department, be as technical as possible, but also remember that this may go to a CFO, CTO, or CEO.
